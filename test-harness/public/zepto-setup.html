Zepto Sandbox Setup

One-time OAuth bootstrap for the Zepto PayTo live sandbox integration

Checking status...
✓
Zepto is connected

The refresh token is stored and the server will auto-refresh access tokens every ~2 hours.

You can now test Zepto in the E2E Testing tab — select Zepto from the APAC region and run Init → Render → Authorize.

1
Authorize the app on Zepto

Click the button below to open the Zepto sandbox authorization page in a new tab. Log in with your Zepto sandbox account and click Authorize.

2
Copy the one-time code

After authorizing, Zepto will display a one-time code on the page (out-of-band flow, urn:ietf:wg:oauth:2.0:oob). Copy it and paste it below.

3
Complete setup

This exchanges the one-time code for a refresh token that the server will store and use to auto-refresh access tokens.

!
Zepto not configured

ZEPTO_CLIENT_ID and ZEPTO_CLIENT_SECRET must be set in test-harness/.env before running the bootstrap flow.

Restart the server after updating .env.
