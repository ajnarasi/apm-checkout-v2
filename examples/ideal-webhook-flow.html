iDEAL — Async Webhook Flow

iDEAL is bank-redirect + async settlement. The user authorizes at their bank, Commerce Hub fires a webhook, the reference server relays it via SSE, and the SDK transitions pending → completed.

1. Create session via merchant backend
2. Initialize SDK and call authorize() — get redirect URL
3. (Simulated) user authorizes at bank
4. Webhook arrives at reference server → SSE → browser SDK
5. PAYMENT_COMPLETED fires
